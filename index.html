hat-trie Documentation

This project is intended to be a fully operational, standards compliant HAT-trie.

Implemented

Here is a list of the major operations that are implemented:

  • begin()
  • clear()
  • count(string)
  • empty()
  • end()
  • erase(string)
  • erase(iterator)
  • exists(string)
  • find(string)
  • insert(record)
  • insert(iterator, iterator)
  • size()
  • swap(hat_set &)
  • forward iteraton and iterator dereferencing

In a hat_set, record is a std::string. In a hat_map, record is a pair<std::string, T>.

Future

Here is a list of major operations that have yet to be implemented:

  • reverse iteration
  • lower_bound(string)
  • upper_bound(string)
  • equal_range(string)
  • match_prefix(string) (an extension)

Usage

Installation

Copy all the headers into a directory in your PATH and include hat_set.h in your project. Some of the headers require stdint.h, which isn't available by default on most Windows platforms. You can find a compatible version of the header on Google.

All classes are defined in namespace stx.

Example

In the vast majority of cases, these classes are used in exactly the same way as their STL counterparts. Just replace set with hat_set. (See the Deviations section for differences.)

 #include "hat_set.h"

 using namespace std;
 using namespace stx;

 int main() {
     hat_set<string> trie;
     trie.insert("hello world");
     cout << *trie.begin() << endl;
     return 0;
 }

Prints hello world.

Extensions

There are a few non-standard extensions to the hat_trie interface:

  • exists(string) -- returns true iff there is a record in the trie with a matching key
  • match_prefix(string) -- returns a set of all strings that have the parameter as a prefix. To be implemented.

Deviations

The hat interface differs from the standard in a few ways:

  • insert(record) -- returns a bool rather than a pair<iterator, bool>. See the HTML documentation for rationale.
  • iterator traversals are unordered. Ordered traversals are in the works.

Testing

The test files in the test/ directory achieve > 95% coverage of hat_trie.h and array_hash.h.
