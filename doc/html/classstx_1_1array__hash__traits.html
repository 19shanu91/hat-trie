Detailed Description

int stx::array_hash_traits::allocation_chunk_size

This value only affects the speed of the insert() operation. When a slot in the array hash is allocated, it is allocated in blocks of this size until there is enough space for a word. In general, higher values use more memory but require fewer memory copy operations. Try to guess how many average characters your strings will use, then multiply that by (hat_trie__traits.burst_threshold / array_hash_traits::slot_count) to get a good estimate for this value.

If you want memory allocations to be exactly as big as they need to be (rather than in block chunks), set this value to 0. This may be your best option in memory-tight situations. Insertion speed is still impressive even with abundant memory copy operations.

Default 32. Must be non-negative.

int stx::array_hash_traits::slot_count

Number of slots in the hash table. Higher values use more memory but may show faster access times.

Default 512. Must be positive.
